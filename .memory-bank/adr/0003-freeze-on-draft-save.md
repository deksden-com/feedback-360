# ADR 0003 — Freeze on first draft save
Status: Draft (2026-03-03)

## Decision
Фиксируем `campaign.locked_at` на первом draft save в любой анкете.
После lock нельзя менять матрицу назначений и веса групп.
После lock можно менять операционные параметры (end_at, напоминания) с audit log.

## Why
- Ранний lock снижает риск “подгонки” матрицы/весов под первые ответы.
- Соответствует требованию: “первый ответ = draft save”.

## Trade-offs / mitigations
- Риск случайного lock (пользователь сохранил 1 поле).
  - Митигация: UI предупреждает, а HR понимает, что после старта лучше завершить матрицу до открытия доступа.
