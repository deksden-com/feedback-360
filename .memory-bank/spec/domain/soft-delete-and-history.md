# Soft delete & history
Status: Draft (2026-03-03)

Зафиксировано:
- Soft delete нужен: `is_active`/`deleted_at` вместо физического удаления.
- Сотрудник может менять подразделение/должность/руководителя во времени.
- В кампанию попадает снапшот на момент старта; после старта снапшот не обновляем.
- UI показывает “последнее” состояние справочника, но исторические кампании используют снапшот.

MVP правила (уточнение):
- Soft-deleted сотрудники/подразделения не попадают в “актуальные” списки выбора (например, при подборе участников/оценщиков).
- Исторические кампании не теряют данные: они читают снапшот и/или исторические записи.
