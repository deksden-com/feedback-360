# Org structure & snapshots
Status: Draft (2026-03-03)

## Org data (MVP)
Оргструктура заводится вручную:
- подразделения и их иерархия,
- принадлежность сотрудников подразделениям,
- отношения “кто кому руководитель”.

Справочник сотрудника (MVP) включает также контакты:
- телефон (для справки/будущих интеграций),
- Telegram account (`telegram_user_id/chat_id`) для будущих уведомлений.

## Snapshot on campaign start
На `campaign.start` делаем снапшот всех участвующих лиц “как было на момент старта”:
- идентификационные поля сотрудника, должность, подразделение(я),
- прямой руководитель (для автогенерации и отчётов),
- базовые флаги/атрибуты, необходимые для результатов и истории.

После старта:
- снапшот **не обновляется**, даже если справочник/оргструктура меняются,
- UI может показывать “текущее” состояние из справочника, но результаты/назначения и отчёты кампании опираются на снапшот.
