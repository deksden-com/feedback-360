# Glossary
Status: Draft (2026-03-03)

Цель: единые термины. Если термин не определён здесь — добавляем сюда, а не “как получится в коде”.

## Core entities
- **Company (тенант)**: отдельная организация в multi-tenant инсталляции. Почти все данные имеют `company_id`.
- **User (Supabase Auth user)**: учётная запись для аутентификации (email + сессии). Один user соответствует одному email и может состоять в нескольких компаниях.
- **Employee (HR-справочник)**: запись сотрудника внутри конкретной Company. Employee и User — разные сущности.
- **Company membership**: связь `user_id ↔ company_id` с ролью доступа (HR admin/HR reader/manager/employee).

## 360 domain
- **Campaign (кампания)**: конкретный процесс 360-оценки в компании со сроками и статусами. Кампания использует зафиксированную версию модели компетенций.
- **Competency model version**: версия модели компетенций (indicators или levels), на которую ссылается кампания.
- **Competency**: компетенция (вопрос/раздел), по которой собираются оценки.
- **Indicator**: утверждение/вопрос со шкалой 1..5 + NA (“не могу оценить”), входящий в компетенцию (для модели indicators).
- **Level**: уровень 1..4 + UNSURE (“затруднился”), описание уровня для компетенции (для модели levels).
- **Rater**: оценивающий (employee), который заполняет анкету.
- **Subject**: сотрудник (employee), которого оценивают.
- **Assignment (назначение)**: связь “кто кого оценивает” в рамках кампании с ролью группы (manager/peers/subordinates/self/other).
- **Questionnaire (анкета)**: один набор ответов “один rater → один subject” в рамках кампании.
- **Draft save**: сохранение черновика анкеты (появляются сохранённые ответы, не обязательно submit).
- **Submit**: финальная отправка анкеты; после submit ответы immutable.

## Results & privacy
- **Anonymity threshold**: минимальное число submitted оценщиков в группе, чтобы показывать агрегаты группе (по умолчанию 3).
- **NA**: “не могу оценить” (не участвует в числовых агрегациях indicators).
- **UNSURE**: “затруднился ответить” (не участвует в числовых агрегациях levels).

## Notifications & ops
- **Outbox**: таблица/очередь исходящих уведомлений (email/telegram), куда записываем “что надо отправить”, а отдельный dispatcher делает фактическую отправку.
- **Idempotency key**: стабильный ключ, по которому мы предотвращаем дубликаты (например, повторная генерация reminders или ретрай отправки).
- **Quiet hours**: окно времени, вне которого уведомления не отправляем (MVP: 08:00–20:00 по таймзоне кампании).
- **Campaign invite**: уведомление, содержащее magic link для входа в систему при старте кампании (MVP email-only).

## AI processing
- **Processed text**: переписанный AI “нейтральным стилем” текст комментариев, предназначенный для employee/manager views.
- **Summary**: краткое резюме processed текста с сохранением ключевых смыслов (используется в результатах).
